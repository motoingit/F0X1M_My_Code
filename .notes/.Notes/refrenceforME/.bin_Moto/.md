Excellent question, Mohit — this formatting style is called a **fenced code block** in Markdown.

Let’s break it down clearly 👇

---

### 🧱 **1. What You’re Seeing**

When you write something like:

````markdown
```markdown
# 📁 file
text content here
````

````
that triple-backtick syntax (```) tells Markdown:  
> “Hey, this is a block of code or formatted text. Don’t treat it like normal prose.”

The word immediately following the opening backticks — here `markdown` — is called a **language identifier**.  
It’s used for **syntax highlighting**.

---

### 🧠 **2. How Syntax Highlighting Works**
If you replace `markdown` with another language name — say `python`, `html`, or `bash` — Markdown renderers (like GitHub, VS Code, Notion, etc.) apply color themes and styles for that language’s syntax.

For example:
```python
def greet(name):
    print(f"Hello, {name}!")
````

→ will highlight keywords (`def`, `print`) in Python colors.

Whereas:

```html
<h1>Hello, World!</h1>
```

→ will colorize as HTML.

And:

```bash
pip install flask
```

→ will appear like shell commands.

---

### 🧾 **3. Supported Examples**

| Language Identifier  | Effect                          |
| -------------------- | ------------------------------- |
| `python`             | Python syntax highlighting      |
| `bash` or `shell`    | Command-line scripts            |
| `json`               | JSON objects formatted          |
| `html`               | HTML colorized                  |
| `css`                | CSS colorized                   |
| `javascript` or `js` | JS colorized                    |
| `markdown`           | Markdown itself shown literally |

---

### ✅ **Pro Tip**

* Use `text` if you **don’t** want any syntax coloring (plain output).
* Use `markdown` if you’re **showing Markdown syntax** itself (like in documentation).
* Use `python`, `bash`, etc., when you want readers to **see colors and semantics** for that language.
