## Big Picture (Executive Summary)

HTML tags are **building blocks** used to structure content on the web.
Every tag falls into a **functional category**. Once you understand the categories, the tags become predictable and easy to master.

Think of HTML as a **document architecture system**, not a programming language.

---

## Core Structure Tags (Mandatory Foundation)

These define the skeleton of every HTML document.

```html
<!DOCTYPE html>
<html>
<head>
<title>My Page</title>
</head>
<body>
</body>
</html>
```

| Tag          | Purpose              |
| ------------ | -------------------- |
| `<!DOCTYPE>` | Declares HTML5       |
| `<html>`     | Root of the document |
| `<head>`     | Metadata container   |
| `<title>`    | Browser tab title    |
| `<body>`     | Visible content      |

👉 **Non-negotiable**: Every HTML page needs these.

---

## Text & Formatting Tags (Content Layer)

Used for readable content.

| Tag               | Use                                |
| ----------------- | ---------------------------------- |
| `<h1>`–`<h6>`     | Headings (importance order)        |
| `<p>`             | Paragraph                          |
| `<br>`            | Line break                         |
| `<hr>`            | Horizontal rule                    |
| `<strong>`        | Strong importance                  |
| `<em>`            | Emphasis                           |
| `<b>` / `<i>`     | Visual only (avoid in modern HTML) |
| `<mark>`          | Highlight                          |
| `<small>`         | Small text                         |
| `<sup>` / `<sub>` | Superscript / Subscript            |

💡 **Rule**: Use tags for **meaning**, not appearance.

| `<ol>` | Ordered list     |
| `<li>` | List item        |

## Links & Media (Connectivity Layer)

This is how the web becomes the web.

| Tag        | Purpose      |
| ---------- | ------------ |
| `<a>`      | Hyperlink    |
| `<img>`    | Image        |
| `<audio>`  | Audio        |
| `<video>`  | Video        |
| `<source>` | Media source |
| `<track>`  | Subtitles    |

⚠️ `<img>` has **no closing tag**.

| Tag         | Role         |
| ----------- | ------------ |

| `<th>`      | Header cell  |
| `<thead>`   | Header group |
| `<tbody>`   | Body group   |
| `<tfoot>`   | Footer group |

Critical for real-world applications.

| Tag          | Function         |
| ------------ | ---------------- |
| `<form>`     | Form container   |
| `<input>`    | Input field      |
| `<textarea>` | Multi-line input |

## Semantic Tags (Modern HTML – Very Important)

These describe **what the content is**, not how it looks.

| Tag            | Meaning             |
| -------------- | ------------------- |
| `<header>`     | Page/section header |
| `<nav>`        | Navigation          |
| `<main>`       | Main content        |
| `<section>`    | Section             |
| `<article>`    | Independent content |
| `<aside>`      | Sidebar             |
| `<footer>`     | Footer              |
| `<figure>`     | Media block         |
| `<figcaption>` | Caption             |

💼 **Corporate truth**: Semantic HTML improves **SEO, accessibility, maintainability**.

---

## Container & Generic Tags

Used when no semantic tag fits.

| Tag      | Purpose          |
| -------- | ---------------- |
| `<div>`  | Block container  |
| `<span>` | Inline container |

🚫 Overusing `<div>` = bad architecture.

---

## Meta & Resource Tags (Head-only)

Used inside `<head>`.

| Tag        | Use                |
| ---------- | ------------------ |

| `<link>`   | External resources |
| `<style>`  | Internal CSS       |
| `<script>` | JavaScript         |
| `<base>`   | Base URL           |
